# :lucide-headset: 資安事件通報與分析

!!! Abstract "快速通報"

    - 另存原始郵件（.eml）、素材。
    - 將原始郵件以附件檔案的方式寄信到 <ssd@ocf.tw>

以下說明可受理的「可通報情境類型」，以及「釣魚、詐騙信通報」的操作重點與蒐證指引，協助您安全、有效地提供線索。

## 可通報的情境類型

### 假網站、惡意網站

- 仿冒政府、學校、醫療、銀行、購物、募款等網站，或短網址跳轉到疑似釣魚頁。
- 通報重點：完整網址（含參數）、截圖、發現時間。如可以取得：DNS/WHOIS 資訊、TLS 憑證指紋或序號。

### 社群平台假帳號、假粉專

- 假冒官方或客服、假投資群組、假抽獎活動。
- 通報重點：帳號或貼文連結、截圖、可疑私訊片段。

### 惡意 App、假 App 散布

- 假冒政府或銀行 App、非官方 APK 下載頁。
- 通報重點：商店頁面或下載頁連結。如可以取得：檔案雜湊值（SHA-256）、簽章/開發者資訊、截圖（請勿執行檔案）。

### 帳號安全異常

- 官方帳號疑遭盜用、異地登入警示、疑似外洩清單（Ptt、Dcard、論壇、Telegram 頻道）出現貴單位資訊。
- 通報重點：來源連結或截圖、受影響平台與帳號 ID、時間。

### DDoS、服務可用性異常線索

- 大量同時無法連線、同區用戶共同回報、DNS 解析異常。
- 通報重點：時間區段、所在地或 ISP。如可以取得：錯誤畫面/代碼、簡要 traceroute/dig 結果。

### 勒索、恐嚇郵件與深偽語音

- 假冒公部門或長官、性勒索信、AI 深偽語音詐騙。
- 通報重點：郵件原始檔（.eml, .msg）、音檔、來電號碼與通話紀錄截圖。

!!! warning "蒐證與安全提醒"

    - 請勿點擊可疑連結、勿啟用巨集或執行未知檔案。
    - 若已取得檔案，請勿開啟。
    - 若涉及即時財損或金流詐騙，請先撥打 165 反詐騙諮詢專線，或視情況報案（110），並同時通報我們協助研判。

## 釣魚、詐騙信通報指南

1. 先行判斷的常見特徵
    - 寄件網域可疑、與官方網域相似但不同（例如字母替換）。
    - 信中要求立即操作（更新帳號、重新登入、付款）並附上連結或附件。
    - 連結顯示的網址與實際導向不一致、使用短網址或奇怪網域。
    - 附件為可執行檔或巨集文件（.exe、.scr、.js、啟用巨集的 Office 檔）。
    - 語氣催促、排版凌亂、語法錯誤或不合常理的請求。
2. 通報時請提供
    - 原始郵件檔：.eml 或 .msg 含完整信標頭與內文（如何打包郵件請參考下一段落的指引）。
    - 截圖：收件畫面、信中連結滑過顯示的實際網址、附件檔名與大小。
    - 基本資訊：收到時間（含時區）、寄件者地址、郵件主旨、您使用的郵件服務/客戶端與版本。
    - 若信內含連結或檔案：請提供連結文字與實際 URL（可右鍵複製），以及檔案雜湊值（SHA-256，勿執行檔案）。
3. 如何打包「原始郵件」
    - **Gmail（網頁版）：**打開該封郵件 → 右上角更多（三點）→ 顯示原始郵件 → **下載原始郵件（.eml）**。
    - **Outlook（Windows 桌面版）：**開啟郵件 → 檔案 → **另存新檔** → 格式選 Outlook 物件（.msg）。如需標頭：檔案 → 內容 → 複製 Internet 標頭。
    - **Outlook on the Web（OWA）：**開啟郵件 → 更多（三點）→ 檢視 → 檢視郵件詳細資料（可複製標頭）。部分版本支援下載 .eml（下載/Download）。
    - **Apple Mail（macOS）：**選取郵件 → 檔案 → **另存新檔…** → 格式選「原始訊息」或 .eml。或於檢視 → 訊息 → 所有標頭。
    - **Mozilla Thunderbird：**選取郵件 → 檔案 → **另存新檔** → 檔案（.eml）。

!!! info "Gmail（網頁版）"

    - Gmail（網頁版）的操作流程可以參考[這個頁面](https://ssd.ocf.tw/help/index.html){target="_blank"}。

## 我們如何處理您的通報

- 收到通報後立即產生案件編號，並進行惡意檔案初步過濾與資料去識別化。
- 專家研判與社群協作：在不涉及個資與敏感營運資訊的前提下，我們會將可公開的威脅指標（如網域、IP、檔案雜湊、樣本特徵、攻擊手法）提供給受信任的資安社群與學研夥伴，共同解析與研究，必要時採行保密協議或限制性分享。
- 跨單位通報與轉介：研判確認後，依權責通知受影響單位，必要時轉介平台業者或主管機關，並同步回饋社群最新處置進度與緩解建議。
- 對外回饋與資訊釋出：處置完成後，提供精簡回饋與可公開之 [IoC（入侵指標，不含個資）](https://www.cloudflare.com/zh-tw/learning/security/what-are-indicators-of-compromise/){target="_blank"}，並定期彙編趨勢觀察與防護建議。視情況將去識別化的威脅指標納入公開或共享的情資系統（如 [STIX/TAXII](https://www.cloudflare.com/zh-tw/learning/security/what-is-stix-and-taxii/){target="_blank"}），協助網路生態系快速阻擋散布。
- 透明與保護：全程遵循個資與法規要求，僅分享最小必要資訊，並妥善保護通報者身分與敏感細節。

## 如何通報

1. 準備好以上情境需要的原始郵件或素材。
2. 將原始郵件、素材以附件檔案的方式寄信到 <ssd@ocf.tw>
